# PRD 전체 기준 미구현/미완료 항목 정리

- 작성일: 2026-05-10
- 기준 문서: `docs/PRD-manual-capture-workbench.md`
- 판정 대상: `webpage-capture-tool` 현재 구현 코드 전체
- 판정 기준:
  - `완료`: 사용자 관점의 흐름이 실제로 동작하고, 저장/복원/로그까지 PRD 계약을 충족함
  - `부분 완료`: 엔진 또는 UI 일부는 있으나 PRD에서 약속한 최종 사용 흐름이 비어 있음
  - `미구현`: 해당 기능을 실행할 사용자 경로 또는 엔진 자체가 없음

## 1. 요약

- `P0/핵심 목표` 기준으로 아직 완료로 보기 어려운 항목이 남아 있다.
- 특히 `출력 프로필 저장/복원`, `비율 옵션`, `고정 비율 크롭`, `레시피 재적용`, `오류 구분 로깅`, `규칙-결과 추적성`이 핵심 공백이다.
- `P1/후속 범위`는 상당수 항목이 아직 시작되지 않았거나 화면 수준에서 비어 있다.
- PRD 밖이지만, 샘플 검증 명령과 실제 워크스페이스 실행 경로가 어긋나는 문서 불일치도 존재한다.

## 2. P0 기준 미완료 또는 부분 완료 항목

| 항목 | PRD 근거 | 상태 | 현재 구현 | 남은 차이 |
|---|---|---|---|---|
| 출력 프로필 저장/복원 | FR-002, Use Case 4, Use Case 5 | 부분 완료 | 내보내기 UI는 `AppState.exportConfig`를 사용하고, 프로젝트 저장은 `project.exportProfiles`를 사용한다. | 실제 사용자가 설정한 채널/출력 경로/파일명 규칙이 프로젝트와 레시피에 일관되게 저장·복원되지 않는다. |
| 프로젝트 재열기 후 작업 세션 복원 | FR-002, FR-050, Use Case 5 | 부분 완료 | 프로젝트 저장/열기 자체는 있다. | 저장 시 포함한 `captureResults`가 `loadProject()` 마이그레이션 경로에서 복원되지 않아 이전 캡처 세트 기반 재작업 흐름이 끊긴다. |
| 비율 옵션 `16:9 / 4:3 / 자유 / 원본 유지` | FR-022 | 미구현 | 캡처 프리셋은 `word / ppt / markdown / custom`만 제공한다. | PRD가 요구한 비율 옵션 자체가 화면과 상태 모델에 없다. |
| 고정 비율 크롭 | FR-040 | 부분 완료 | 자유 크롭은 있다. | `16:9`, `4:3`, `원본 유지` 같은 비율 고정 크롭 UI와 규칙 모델이 없다. |
| 레시피 반복 적용 | FR-051, 제품 목표 `레시피 저장/재적용` | 부분 완료 | 레시피 저장/불러오기와 배치 적용 버튼은 있다. | 배치 적용 시 `crop`, `resize`를 제외하고 적용하며, DOM 규칙과 이미지 규칙을 하나의 재실행 흐름으로 여러 대상에 온전히 재적용하지 못한다. |
| 오류 유형 구분 표시 | FR-070 | 부분 완료 | DOM 규칙 경고 로그와 일반 실패 로그는 일부 존재한다. | UI 기준으로 `DOM 규칙 실패`, `네비게이션 실패`, `캡처 실패`가 별도 상태/카테고리로 정리되지 않는다. 현재는 대부분 일반 `error` 또는 raw stdout 문자열로 보인다. |
| 규칙-결과 추적 로그 | FR-072, 비기능 요구 `추적성` | 부분 완료 | 캡처 엔진 내부에는 `domResults`와 결과 배열이 있다. | 어떤 규칙이 어떤 산출 파일에 적용됐는지를 사용자에게 추적 가능하게 남기지 않는다. renderer는 `capture-result`를 기대하지만 실제 emit이 없다. |

## 3. P1 및 후속 범위 기준 미구현 항목

### 3.1 프로젝트 / 캡처 / DOM

| 항목 | PRD 근거 | 상태 | 메모 |
|---|---|---|---|
| 최근 프로젝트 목록 확인 및 재실행 | FR-003 | 미구현 | 화면 렌더 함수는 있지만 `recentProjects`를 채우거나 영속화하는 로직이 없다. |
| 채널별 안전 영역 가이드 프리뷰 | FR-023 | 미구현 | PPT/Word용 안전 영역 오버레이나 가이드 UI가 없다. |
| DOM 적용 전/후 미리보기 | FR-033 | 미구현 | DOM 편집 화면은 규칙 표 중심이며 웹 프리뷰 비교 흐름이 없다. |
| 사용자 CSS 주입 | FR-034 | 미구현 | 입력 필드, 상태 모델, core 실행 경로 모두 없다. |
| 속성 치환(`src`, `href`, `value`) | PRD 메뉴 구조, Phase 2 범위 | 미구현 | 현재 DOM 규칙 타입은 `replaceText`, `hide`만 지원한다. |
| selector hover / 클릭 기반 선택 | PRD 화면/UX 방향 | 미구현 | selector를 직접 입력하는 방식만 있고 미리보기에서 요소를 집는 흐름은 없다. |

### 3.2 이미지 편집 / 배치

| 항목 | PRD 근거 | 상태 | 메모 |
|---|---|---|---|
| 텍스트 라벨 | FR-046, 메뉴 구조 | 미구현 | 이미지 편집 도구 목록에 없다. |
| 화살표 | FR-046, 메뉴 구조 | 미구현 | 이미지 편집 도구 목록에 없다. |
| 강조 박스 / 스텝 번호 | FR-046, 제품 목표 `간단한 주석 편집` | 미구현 | 현재 `box`는 마스킹용 단색 박스이며 주석/강조 박스와는 다르다. |
| 페이지별 예외 규칙 덮어쓰기 | FR-052 | 미구현 | 프로젝트/레시피 모델에 per-page override 구조가 없다. |
| 실패 selector 복사 | FR-071 | 미구현 | 실패 URL 복사만 있고 실패 selector 복사 기능은 없다. |
| 다중 캡처 세션 관리 | Phase 2 범위 | 미구현 | 단일 현재 세션 중심 상태만 존재한다. |
| 프로젝트 단위 작업 이력 | Phase 2 범위, 메뉴 구조 `작업 기록` | 미구현 | 하단 로그는 있으나 프로젝트 이력/최근 출력물/세션 기록으로 남기지 않는다. |

## 4. UX / 비기능 요구사항 기준 공백

| 항목 | PRD 근거 | 상태 | 메모 |
|---|---|---|---|
| 원본 / 적용 후 전환 프리뷰 | UX 세부 요구 `프리뷰` | 미구현 | 이미지 편집은 현재 상태를 덧그려 보이지만, 원본과 결과를 토글하는 비교 UX는 없다. DOM 쪽은 더 직접적인 프리뷰 자체가 없다. |
| 배치 진행 상태 가시성 | UX 세부 요구 `배치 피드백` | 부분 완료 | 로그 텍스트와 종료 후 성공/실패 개수는 보이지만, 별도 진행 바나 구조화된 진행률 UI는 없다. |
| 접근성 강화 | UX 세부 요구 `접근성` | 부분 완료 | 일부 키보드 단축키와 `:focus` 스타일은 있으나, 화면 전반의 키보드 중심 흐름과 명확한 포커스 설계가 PRD 수준으로 정리돼 있지 않다. |
| 결과물 추적성 | 비기능 요구 `추적성` | 부분 완료 | 어떤 원본·규칙·출력 프로필에서 산출물이 나왔는지 완결된 manifest/로그 연결이 부족하다. |
| 민감정보 로그/중간 산출물 정책 | 비기능 요구 `보안`, 리스크 `운영 데이터 노출` | 미구현 | 로그 마스킹 정책, 임시 파일 정리 전략, 사용자 데이터 노출 방지 정책이 코드/문서에 정리돼 있지 않다. |
| 성공 지표 측정 수단 | 성공 지표 섹션 | 미구현 | 재사용률, 실패 복구 효율, 출력 일관성 등을 측정할 수단이 없다. |

## 5. 코드 기준 근거 메모

- 출력 프로필 저장 경로와 실제 UI 상태가 분리돼 있다.
  - `apps/electron/renderer/export-screen.js`
  - `apps/electron/renderer/shared-state.js`
  - `apps/electron/renderer/project-screen.js`
- 프로젝트 재열기 시 캡처 결과가 사라진다.
  - 저장: `apps/electron/renderer/project-screen.js`
  - 로드 마이그레이션: `packages/core/src/recipe-store.js`
- 비율 옵션과 고정 비율 크롭 UI가 없다.
  - `apps/electron/renderer/index.html`
  - `apps/electron/renderer/image-screen.js`
- 레시피 배치 적용 시 `crop`, `resize`가 제외된다.
  - `apps/electron/renderer/batch-screen.js`
- renderer는 `capture-result`를 기대하지만 core는 결과를 이벤트로 내보내지 않는다.
  - 기대: `apps/electron/renderer/renderer.js`
  - 실제 캡처 실행: `packages/core/src/screenshot-runner.js`

## 6. 보완 우선순위 제안

1. `출력 프로필 저장/복원`과 `프로젝트 세션 복원`을 먼저 맞춘다.
2. `FR-022`, `FR-040`에 해당하는 비율 옵션과 고정 비율 크롭을 추가한다.
3. `FR-051`, `FR-070`, `FR-072`를 보완해서 레시피 반복 작업과 실패 분석 흐름을 완성한다.
4. 그 다음 P1 항목인 `DOM 프리뷰`, `사용자 CSS`, `주석 도구`, `최근 프로젝트/작업 이력`으로 확장한다.
5. 마지막으로 Phase 2/3 항목인 `docx/pptx`, `시각 회귀`, `공유 프리셋`을 별도 마일스톤으로 분리한다.

## 7. 부록: PRD 외 추가 발견사항

- `docs/contributor-guide.md`에 적힌 샘플 검증 명령 `npm run cli -- --file docs/sample-files/sample.xlsx` 는 현재 워크스페이스 실행 기준으로 실패한다.
- 절대 경로로 주면 CLI 샘플 캡처는 정상 동작한다.
- `lint` 스크립트가 renderer 분리 파일과 일부 core 신규 파일을 모두 검사하지는 않는다.
