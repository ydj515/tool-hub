## 1. 제품 개요

### 한 줄 설명

웹페이지 기반 사용자 매뉴얼 제작을 위해, **캡처 전 DOM 치환**과 **캡처 후 마스킹/주석 편집**, **채널별 산출물 내보내기**를 하나의 워크플로로 제공하는 데스크톱 도구입니다.

### 배경

현재 사용자 매뉴얼 제작은 다음과 같은 흐름으로 이뤄지는 경우가 많습니다.

1. 실제 또는 개발 환경의 웹페이지를 연다.
2. 필요한 화면을 수동으로 캡처한다.
3. 모킹 데이터가 보이는 영역을 포토샵, Preview, 파워포인트 등으로 가린다.
4. 텍스트를 다시 얹거나 강조 박스를 넣는다.
5. Word, PPT, Markdown 문서에 맞게 이미지 크기를 다시 조절한다.
6. 같은 작업을 다른 화면에서도 반복한다.

이 과정은 반복량이 많고, 동일한 수정 규칙을 재사용하기 어렵고, 캡처 전 단계에서 더 깔끔하게 처리할 수 있는 항목까지 이미지 편집으로 해결하게 만드는 문제가 있습니다.

`webpage-capture-tool`은 이미 Playwright 기반 캡처와 Electron UI를 갖고 있으므로, 이 프로젝트를 단순 스크린샷 도구에서 **매뉴얼 제작용 캡처 워크벤치**로 확장할 수 있습니다.

## 2. 문제 정의

### 현재 문제

| 문제 | 설명 |
|---|---|
| 반복적인 수작업 | 화면마다 크롭, 리사이즈, 블러, 박스 가리기, 텍스트 삽입을 반복해야 함 |
| 원본 품질 저하 | 캡처 전에 가릴 수 있는 데이터도 이미지 위에 덮어써서 결과물이 부자연스러워짐 |
| 일관성 부족 | Word용, PPT용, Markdown용 이미지 크기와 여백이 매번 달라짐 |
| 재사용성 부족 | 같은 서비스의 여러 화면에 동일한 마스킹/치환 규칙을 재적용하기 어려움 |
| 협업 어려움 | 어떤 화면에서 어떤 규칙을 썼는지 추적하기 어렵고, 팀 내 표준화가 어려움 |
| 실패 복구 비용 | 캡처 실패, 잘못된 치환, 잘못된 마스킹을 다시 처음부터 반복해야 함 |

### 현재 `webpage-capture-tool`의 한계

현재 도구는 다음 역할에는 강합니다.

- URL 목록 기반 전체 페이지 캡처
- 출력 경로 지정
- 실패 URL 재실행
- 대기 시간, 헤드리스, 중복 제거 등 실행 옵션 관리

반면 아래 기능은 아직 비어 있습니다.

- 캡처 전 DOM 텍스트 치환
- selector 기반 요소 숨기기
- 부분 캡처와 비율 프리셋
- 캡처 후 블러/박스 마스킹
- 주석/번호/강조 도구
- Word / PPT / Markdown 대상별 내보내기
- 작업 레시피 저장 및 재사용

## 6. 핵심 사용 시나리오

### Use Case 1: 로그인 화면의 사용자명 치환 후 캡처

1. 사용자가 단일 URL 또는 목록 파일을 입력한다.
2. `#user-name` selector의 텍스트를 `홍길동`으로 치환하는 규칙을 추가한다.
3. 뷰포트를 `Word 기본 (1440x1024)` 프리셋으로 선택한다.
4. 캡처를 실행한다.
5. 결과 이미지에는 실제 테스트 계정 대신 지정한 텍스트가 반영된다.

### Use Case 2: 개인 정보 영역을 블러로 가리고 내보내기

1. 캡처된 이미지에서 이메일/전화번호가 보이는 영역을 선택한다.
2. `블러` 도구로 처리한다.
3. `PPT 16:9` 출력 프로필로 내보낸다.
4. 결과 이미지는 슬라이드에 맞는 비율과 크기로 정리된다.

### Use Case 3: 배너/알림 박스를 DOM에서 숨기고 전체 흐름 캡처

1. 사용자가 `selector 숨기기` 규칙에 `.toast`, `.notice-banner`를 등록한다.
2. 여러 URL을 배치 실행한다.
3. 공통 배너 없이 깔끔한 매뉴얼용 화면이 일괄 생성된다.

### Use Case 4: Word / PPT / Markdown 채널별 패키지 생성

1. 하나의 캡처 세트에 대해 `Word`, `PPT`, `Markdown` 3개 출력 프로필을 선택한다.
2. 도구는 이미지 리사이즈와 파일 네이밍을 각 채널 규격에 맞게 수행한다.
3. Markdown의 경우 `.md` 초안과 상대경로 이미지 링크를 함께 생성한다.
4. Word/PPT의 경우 각 채널용 이미지 세트와 캡션 메타데이터 패키지를 생성한다.

### Use Case 5: 다음 릴리즈에서도 같은 규칙 재사용

1. 사용자가 `서비스 A - 회원관리 매뉴얼` 프로젝트를 저장한다.
2. 저장된 프로젝트에는 URL 목록, DOM 치환 규칙, 마스킹 규칙, 출력 프리셋이 포함된다.
3. 다음 버전 매뉴얼 작업 시 프로젝트를 열어 일부 URL만 바꿔 재실행한다.

## 7. 제품 원칙

1. **DOM에서 해결할 수 있는 것은 DOM에서 먼저 해결한다.**
   이미지 후처리보다 캡처 전 치환/숨기기가 우선이다.

2. **매뉴얼 제작에 필요한 기능만 빠르게 제공한다.**
   범용 이미지 편집기처럼 확장하지 않는다.

3. **반복 가능한 레시피를 자산으로 만든다.**
   한 번 한 편집은 다음에도 다시 써야 한다.

4. **채널별 출력은 결과물 품질 기준으로 설계한다.**
   같은 이미지를 무조건 재사용하는 대신, Word / PPT / Markdown의 소비 맥락을 반영한다.

5. **현재 모노레포 구조를 최대한 재사용한다.**
   `packages/core`는 실행 엔진, `apps/electron`은 워크벤치 UI 역할을 맡는다.

## 8. 메뉴 구조 / 정보 구조

### 상위 메뉴 구조

```text
프로젝트
  - 최근 작업
  - 새 프로젝트
  - 프로젝트 저장

캡처
  - 단일 URL
  - 목록 파일(xlsx/csv/txt)

DOM 편집
  - 텍스트 치환
  - 속성(src/href/value) 치환
  - selector 숨기기
  - 사용자 CSS 주입

이미지 편집
  - 크롭
  - 비율 고정 리사이즈
  - 블러
  - 박스 가리기
  - 텍스트 라벨
  - 화살표 / 강조 박스 / 스텝 번호

배치 작업
  - 레시피 저장
  - 레시피 불러오기
  - 여러 페이지 일괄 적용
  - 실패 항목 재실행

내보내기
  - Word
  - PPT
  - Markdown
  - 파일명 규칙
  - 출력 경로

작업 기록
  - 최근 실행 로그
  - 결과 미리보기
  - 실패 내역
```

### 화면 레이아웃 제안

| 영역 | 역할 |
|---|---|
| 좌측 사이드바 | 프로젝트, 캡처 목록, 저장된 레시피, 최근 출력물 |
| 중앙 메인 | 웹페이지 미리보기 또는 캡처 이미지 캔버스 |
| 우측 패널 | 현재 선택 도구의 속성 편집 패널 |
| 하단 로그 영역 | 실행 로그, 실패 URL, 내보내기 상태 |

### 주요 메뉴 흐름

1. `캡처`에서 URL과 뷰포트, 캡처 범위를 정한다.
2. `DOM 편집`에서 치환/숨기기 규칙을 입력한다.
3. 캡처 후 `이미지 편집`에서 블러, 박스, 주석을 보완한다.
4. `배치 작업`에서 레시피를 저장하거나 여러 화면에 반복 적용한다.
5. `내보내기`에서 Word/PPT/Markdown 채널별 패키지를 생성한다.

## 11. 채널별 출력 정책

### 11.1 Markdown 출력

생성물:

- `manual.md`
- `images/` 폴더
- `manifest.json`

정책:

- 상대 경로 기반 이미지 링크 사용
- alt 텍스트와 기본 캡션 포함
- 번호형 섹션 템플릿 제공

예시:

```md
## 1. 로그인 화면

![로그인 화면](./images/001_login.png)

사용자 ID와 비밀번호를 입력한 뒤 `로그인` 버튼을 클릭합니다.
```

### 11.2 Word 출력

MVP 생성물:

- `word-assets/` 폴더
- 채널 최적화 PNG 세트
- `captions.csv`
- `manifest.json`

정책:

- A4 문서 삽입을 고려한 기본 너비 프리셋 제공
- 이미지 하단 캡션용 텍스트 분리
- Word에 바로 붙여 넣기 좋은 파일명 순서 유지

후속 확장:

- `.docx` 초안 자동 생성
- 목차/섹션 템플릿 지원

### 11.3 PPT 출력

MVP 생성물:

- `ppt-assets/` 폴더
- 16:9 슬라이드 기준 PNG 세트
- `slides.csv`
- `manifest.json`

정책:

- 슬라이드 안전 영역을 고려한 비율 프리셋 제공
- 한 슬라이드 한 장 이미지 기본 전략
- 제목/설명 텍스트를 메타데이터로 분리

후속 확장:

- `.pptx` 초안 자동 생성
- 표지/섹션 슬라이드 템플릿 지원

## 20. 결론

`webpage-capture-tool`은 이미 캡처 실행기 역할을 수행하고 있으므로, 여기에 **DOM 치환 + 매뉴얼용 후처리 + 채널별 내보내기**를 붙이면 사용자 매뉴얼 제작 흐름에 직접적인 생산성 향상을 줄 수 있습니다.

초기 버전은 다음 네 가지를 중심으로 잡는 것이 가장 현실적입니다.

1. 캡처 전 DOM 치환/숨기기
2. 캡처 후 블러/박스/크롭
3. 레시피 저장/재적용
4. Word / PPT / Markdown 출력 프로필

이 범위는 현재 모노레포 구조와도 잘 맞고, 향후 `.docx` / `.pptx` 초안 생성이나 팀용 프리셋 공유로 자연스럽게 확장할 수 있습니다.
